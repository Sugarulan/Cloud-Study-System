## 2. 后端模块结构

采用 **多模块 Maven 工程**，按业务域拆分：

```
gac-lms-dependencies          ← 仅 BOM，统一版本
gac-lms-common                ← 公共：异常、响应包装、常量、工具
gac-lms-infrastructure        ← 基础设施：MySQL/Redis/MinIO/Security
gac-lms-ai                    ← AI 适配层（Provider 接口）
gac-lms-module-evaluation     ← 3.3.6  评卷
gac-lms-module-grade          ← 3.3.7  成绩
gac-lms-module-exam-taking    ← 3.3.8  在线作答
gac-lms-module-self-test      ← 3.3.9  个人测评
gac-lms-module-knowledge      ← 3.3.10 知识管理
gac-lms-app                   ← 启动模块（聚合）
```

## 3. 关键设计决策

### 3.1 在线作答的"暂存 + 续答"

- 答卷数据存 **Redis Hash**，TTL 2 小时
- Key 格式：`exam:taking:v2:{examId}:{userId}`
- 字段：题目 ID → 答案 JSON（含版本号）
- 提交时用 `WATCH/MULTI` 防止并发覆盖

### 3.2 AI 评卷架构

```
┌─────────────┐    ┌──────────────┐    ┌─────────────────┐
│ 评卷 Service │───▶│ AI Provider  │───▶│ 企业内网大模型   │
└─────────────┘    │  (接口层)     │    │  (REST API)      │
                   └──────┬───────┘    └─────────────────┘
                          │
                          ▼
                   ┌──────────────┐
                   │ Mock Provider│  ← 降级方案
                   └──────────────┘
```

- **Provider 接口**：统一的 `invoke(prompt)` / `invokeStream(prompt)`
- **Resilience4j 熔断**：连续失败 5 次 → 切到 Mock
- **降级策略**：AI 评卷失败 → 标记为待人工评卷

### 3.3 知识库状态机

```
DRAFT ──submit──▶ PENDING ──approve──▶ PUBLISHED
                          └──reject───▶ REJECTED
PUBLISHED ──archive──▶ ARCHIVED
PUBLISHED ──update──▶ DRAFT（新版本）
```

## 4. 数据库选型

- **MySQL 8** —— 主存储，支持 JSON 字段（题目答案、错题解析）
- **Redis 7** —— 会话、答题暂存、限流计数
- **MinIO** —— 文档、图片、导出文件

## 5. 安全设计

- **Spring Security + JWT**：无状态认证
- **RBAC**：账号 / 角色 / 权限 三级
- **数据权限**：行级（按部门）+ 字段级（敏感字段脱敏）
- **接口防重放**：基于 nonce + timestamp
- **SQL 注入 / XSS**：参数化查询 + HTML 转义
